# GrantAI 운영 SLA (서비스 수준 협약서)
## Phase 1 컨시어지 운영 프로토콜

---

### 📋 운영 개요
- **서비스**: 회사소개서 PDF → HWP 사업계획서 초안 + 정부지원금 매칭
- **가격**: 5만 원 (선착순 10개사 한정)
- **운영자**: 대표 직접 운영 (1인 컨시어지)
- **수신 채널**: 카카오톡 오픈채팅 + 이메일 병행

---

### ⏱️ 단계별 SLA

| 단계 | 트리거 | 운영자 행동 | SLA |
|------|--------|-------------|-----|
| **1. 결제 확인** | Stripe/토스 결제 알림 | 확인 메시지 발송 | **즉시 (10분 내)** |
| **2. PDF 수신** | 고객이 PDF/텍스트 전송 | "접수 확인 + 3시간 내 완료" 안내 | **10분 이내** |
| **3. AI 초안 생성** | PDF 수신 후 | government_reviewer_prompt + Claude API로 초안 생성 | **60분 이내** |
| **4. 검수** | 초안 완성 후 | 오탈자, 수치 일관성, HWP 포맷 확인 | **30분 이내** |
| **5. 발송** | 검수 완료 후 | HWP 파일 + 공고 리스트 카카오/이메일 발송 | **PDF 수신 후 3시간 이내** |
| **6. 팔로우업** | 발송 후 D+3 | "신청 결과 어떠셨나요?" + 피드백 요청 | D+3 오전 10시 |

### 📝 초안 생성 절차 (step-by-step)

#### Step 1: PDF 분석
```
1. 받은 PDF를 Claude Sonnet에 업로드
2. 다음 프롬프트로 핵심 정보 추출:
   "이 회사소개서에서 다음 정보를 추출하라:
   - 기업명, 업종, 설립일
   - 주요 제품/서비스 (최대 3개)
   - 핵심 기술/경쟁우위
   - 타겟 시장 규모 (언급된 경우)
   - 팀 구성 (언급된 경우)
   - 현재 매출/투자 상황 (언급된 경우)"
```

#### Step 2: 공고 매칭
```
1. K-Startup (www.k-startup.go.kr) → 현재 접수 중인 공고 확인
2. Bizinfo (www.bizinfo.go.kr) → 귀사 업종 필터링
3. 추출된 기업 정보 vs 공고 조건 비교
4. 적합도 상위 3개 선정 (마감일, 지원금액, 자격요건 기준)
```

#### Step 3: HWP 초안 생성
```
government_reviewer_prompt.md의 시스템 프롬프트 사용
+ 해당 공고의 사업계획서 양식 템플릿 적용
→ Claude로 초안 텍스트 생성
→ HWP 템플릿에 붙여넣기 (수동, ~15분)
→ 표/차트 삽입
→ 마지막 검토
```

### 🚦 Go/No-Go 검증 지표

| 기준 | 액션 |
|------|------|
| 10건 결제 달성 | → **GO** Phase 2 자동화 투자 시작 |
| 3건 결제 (1주일) | → 계속 운영 + 피드백 기반 카피 개선 |
| 0건 (1주일) | → 가격 A/B 테스트 (3만 원 vs 7만 원) + 채널 피벗 |
| 환불 요청 2건+ | → 품질 검토 + 프롬프트 개선 |
